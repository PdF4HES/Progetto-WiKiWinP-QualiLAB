# A Rust-to-Lean Verification Pipeline with AI Provers: An Experience Report

**Autori:** Natalia Klaus, Palina Tolmach, Juan Conejero · **2026** · [ArXiv](https://arxiv.org/abs/2605.30106) · [PDF](https://arxiv.org/pdf/2605.30106)

## Abstract

We describe a verification pipeline that takes production Rust cryptographic code and produces machine-checked correctness proofs in Lean 4. The pipeline combines three components: symbolic extraction tools (Charon and Aeneas, or Hax) that lift Rust into Lean 4; formal cryptographic specification libraries (ArkLib and CompPoly) that provide the mathematical targets; and AI provers (Aristotle from Harmonic AI and Aleph from Logical Intelligence) that close the resulting proof obligations. Every proof is checked by the Lean kernel, so AI output cannot compromise soundness. The paper describes the architecture, walks through a running example, reports which classes of proof obligations AI closed and which required manual work.
